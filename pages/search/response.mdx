# Response shape

The search endpoint always returns the same shape. We do not vary fields
by query or instrument type — empty values are present-but-null.

## Top-level

```json
{
  "success": true,
  "data": [ { /* Result */ } ],
  "timestamp": "2026-06-06T03:45:11.000Z"
}
```

| Field       | Type     | Notes                                                |
| ----------- | -------- | ---------------------------------------------------- |
| `success`   | boolean  | `true` if search succeeded.                          |
| `data`      | Result[] | Up to `limit` results.                               |
| `timestamp` | string   | ISO-8601 timestamp of when the results were generated. |

<Callout type="info" emoji="🔍">
  `total` is the count *before* `limit`/`offset` are applied. If you
  paginate, divide by `limit` to get the page count.
</Callout>

## `Result`

```json
{
  "id": 334562,
  "canonicalSymbol": "NSE:RELIANCE",
  "wsSubscribeUirId": 334562,
  "symbol": "RELIANCE",
  "name": "RELIANCE",
  "exchange": "NSE",
  "segment": "NSE",
  "instrumentType": "EQ",
  "assetClass": "equity",
  "isDerivative": false,
  "lotSize": 1,
  "tickSize": 10,
  "priceStatus": "stale",
  "last_price": null,
  "change": null,
  "pchange": null,
  "streamProvider": "falcon"
}
```

| Field             | Type    | Notes                                                              |
| ---------------- | ------- | ------------------------------------------------------------------ |
| `id`             | integer | **Unique instrument ID (UIR).** Use this for subscription.        |
| `canonicalSymbol`| string  | `EXCHANGE:SYMBOL` format. Display-only.                               |
| `wsSubscribeUirId`| integer | Same as `id`. Use for WebSocket subscription.                       |
| `symbol`         | string  | Trading symbol (e.g. "RELIANCE").                                  |
| `name`           | string  | Human-readable name.                                              |
| `exchange`       | string  | `NSE`, `BSE`, `MCX`, `CDS`, `NFO`, `BFO`.                          |
| `segment`        | string  | Exchange segment (same as exchange for equities).                   |
| `instrumentType` | string  | `EQ`, `IU`, `FUT`, `CE`, `PE`, `MCX`, `CURRENCY`.                  |
| `assetClass`     | string  | `equity`, `derivative`, `commodity`, `currency`.                   |
| `isDerivative`   | boolean | Whether this is a derivative instrument.                          |
| `lotSize`        | integer | Lot size (derivatives). 1 for cash.                                  |
| `tickSize`       | number  | Smallest price increment.                                          |
| `priceStatus`    | string  | `live`, `stale`, `closed`, `halted`, `pre_open`. See below.        |
| `last_price`     | number? | Most recent trade. `null` if no quote available.                   |
| `change`         | number? | Absolute change (`last_price - close`). `null` if no quote.          |
| `pchange`       | number? | Percentage change. `null` if no quote.                              |
| `streamProvider` | string  | Data provider: `falcon`, `vayu`, `drift`, `atlas`.                  |

## When `priceStatus` matters

| `priceStatus`  | Meaning                                                              |
| -------------- | -------------------------------------------------------------------- |
| `live`         | The platform has a current quote. Tick feed is active.               |
| `pre_open`     | The instrument exists and has a pre-open session but no live ticks.  |
| `closed`       | The market for this instrument is closed for the day (or expired).   |
| `halted`       | Trading is paused. The platform will resume ticks when trading does. |
| `delisted`     | The contract has been delisted. It is in the catalogue for historical reference only. |

<Callout type="info" emoji="📋">
  When the `priceStatus` is not `live`, `last_price`, `change`,
  `pchange` are all `null`. Render a "no quote" state
  — don't render `0`.
</Callout>

## Fields that can be null/absent

| Field          | When null/absent                                              |
| -------------- | ------------------------------------------------------------- |
| `name`         | The catalogue doesn't have a long-form name.                  |
| `lotSize`      | Only meaningful for derivatives.                              |
| `tickSize`     | Only meaningful for instruments with a tick size.            |
| `last_price`   | No current quote.                                             |
| `change`       | No current quote.                                             |
| `pchange`      | No current quote.                                             |
| `streamProvider` | Not available for this instrument.                           |
| `isin`        | Not an equity.                                                |

## A note on `name` vs `canonical`

`canonical` is the trading symbol (the round-trip identifier). `name`
is the company or contract name. For NIFTY options, `canonical` is
`NFO:NIFTY 23500 CE 28 NOV 24` and `name` is `NIFTY 23500 CE 28 NOV 24`
(typically the same string; the contract has no "company" in the way
RELIANCE does).

Always render `name` in the UI. Use `canonical` only for things like the
URL bar.

## Example: a search for "RELIANCE"

```bash
curl "https://marketdata.vedpragya.com/api/search?q=RELIANCE&exchange=NSE" \
  -H "x-api-key: $STREAMS_KEY"
```

```json
{
  "query": "RELIANCE",
  "total": 2,
  "results": [
    {
      "uirId": 114996,
      "canonical": "NSE:RELIANCE",
      "name": "Reliance Industries Limited",
      "exchange": "NSE",
      "instrumentType": "EQUITY",
      "lotSize": 1,
      "tickSize": 0.05,
      "priceStatus": "live",
      "lastPrice": 2347.10,
      "change": 6.00,
      "changePct": 0.26,
      "volume": 12403201,
      "isin": "INE002A01018"
    }
  ],
  "facets": { "exchange": { "NSE": 1, "BSE": 1 }, "type": { "EQUITY": 2 } },
  "took": 7
}
```

The `BSE:RELIANCE` row is hidden by the `exchange=NSE` filter but
appears in the facet count.
