# Query parameters

```
GET /api/search?q=RELIANCE&exchange=NSE&type=EQUITY&limit=10
```

## Field reference

| Name        | Type    | Required | Default     | Notes                                                                |
| ----------- | ------- | -------- | ----------- | -------------------------------------------------------------------- |
| `q`         | string  | Yes      | â€”           | The user query. Already-prefix and fuzzy are both fine.              |
| `exchange`  | string  | No       | (any)       | Comma-separated. Allowed: `NSE`, `BSE`, `MCX`, `CDS`, `NFO`, `BFO`. |
| `type`      | string  | No       | (any)       | `EQUITY`, `INDEX`, `FUT`, `CE`, `PE`, `COMMODITY`, `CURRENCY`.       |
| `limit`     | int     | No       | `10`        | 1 to 50.                                                             |
| `offset`    | int     | No       | `0`         | 0 to 10000. Pagination, not cursor.                                  |
| `sort`      | string  | No       | `relevance` | `relevance`, `name`, `volume`, `change_pct`.                        |
| `priceStatus` | string | No      | `any`       | `live`, `closed`, `any`. Filters by venue status.                   |
| `include`   | string  | No       | â€”           | `ohlcv` returns today's open/high/low/close/volume alongside.        |

<Callout type="info" emoji="đź’ˇ">
  `include=ohlcv` adds a few KB per row; skip it on the typeahead and use
  [`/api/instruments/ltp`](/instruments/ltp-snapshot) for the LTP enrichment
  on selection.
</Callout>

## Examples

### Typeahead for NSE equities only

```bash
curl "https://marketdata.vedpragya.com/api/search?q=REL&exchange=NSE&type=EQUITY&limit=5" \
  -H "x-api-key: $STREAMS_KEY"
```

### Search the option chain by underlying

```bash
curl "https://marketdata.vedpragya.com/api/search?q=NIFTY+23500&exchange=NFO" \
  -H "x-api-key: $STREAMS_KEY"
```

### Sort by top-gainers

```bash
curl "https://marketdata.vedpragya.com/api/search?q=&exchange=NSE&type=EQUITY&sort=change_pct&limit=20" \
  -H "x-api-key: $STREAMS_KEY"
```

<Callout type="warning" emoji="âš ď¸Ź">
  `q` is technically optional when `sort` is set, but the typeahead-style
  search requires a non-empty `q`. If you need a "show me top movers"
  screen, use `q=` (empty) plus a `sort` and a tight `limit`.
</Callout>

## Error modes

| Code | Meaning                                       | What to do                       |
| ---- | --------------------------------------------- | -------------------------------- |
| 400  | `q` missing and no `sort` set                 | Pass at least one.               |
| 401  | API key missing or invalid                    | Re-issue from admin panel.       |
| 429  | Rate limit hit                                | Honor `Retry-After`.             |
| 5xx  | Index unavailable                             | Retry with backoff; report if persistent. |
