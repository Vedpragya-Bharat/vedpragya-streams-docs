# Instruments

The "instruments" surface is everything you need to *resolve* a symbol into
a UIR id and *quote* it without opening a WebSocket. It is the read-only,
REST-only counterpart to the live stream.

## Identifier discipline

There is exactly **one** identifier your code should care about: the UIR
id. It is stable. It is numeric. It works on every endpoint. The search
result, the live feed, the watchlist, the snapshot, the OHLCV — all of them
take and return UIR ids.

You will see two other identifier-shaped things in some responses:

- `canonical` — the `EXCHANGE:SYMBOL` form (e.g. `NSE:RELIANCE`,
  `NFO:NIFTY 23500 CE 28 NOV 24`). Human-readable, round-trips to a UIR id.
- `providerToken` — the underlying provider's instrument token. This is
  present only in a few admin/debug surfaces. **Do not depend on it.**

## Don't mix spaces

The platform maintains two distinct identifier spaces:

1. **UIR space** (public, what you use)
2. **Provider-token space** (private, internal)

The provider's token is a 32-bit integer assigned by the upstream market
data vendor. The UIR id is a 64-bit integer assigned by the platform.
The search API returns UIR ids; the live WebSocket accepts UIR ids. The
two are never equal for the same contract.

If you ever see a `providerToken` leaking into your UI, it's a bug — file
it.
