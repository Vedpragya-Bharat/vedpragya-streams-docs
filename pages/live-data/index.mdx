# Live data

The live gateway is a Socket.IO WebSocket at `wss://stream.vedpragya.com`.
It speaks a tiny, stable protocol. There is **one** message type for
subscribing, **one** event name for ticks, and **one** for subscription
acknowledgements. Everything else is metadata.

## What you get

- **Push-based ticks.** Average tick-to-client latency: 8–30 ms.
- **One socket per browser tab.** Multiplexed subscriptions.
- **Up to 1,000 instruments per socket.** More on request.
- **Auto-reconnect with replay.** We replay the last 60 seconds of ticks
  on reconnect.
- **Daily session refresh is invisible.** The platform handles the
  upstream market data provider's daily access-token refresh. You keep one
  long-lived API key.

## The five-minute version

  <Tab>
    ```js
    import { io } from 'socket.io-client';

    const socket = io('https://stream.vedpragya.com', {
      extraHeaders: { 'x-api-key': process.env.STREAMS_KEY },
      transports: ['websocket'],
    });

    socket.on('connect', () => {
      socket.emit('subscribe', {
        channel: 'market_data',
        instruments: [114996, 26026],
      });
    });

    socket.on('subscription_confirmed', (ack) => {
      console.log('subscribed to', ack.instruments);
    });

    socket.on('market_data', (tick) => {
      console.log(tick.uirId, tick.lastPrice);
    });
    ```

## What the platform hides

You do not need to know which market data provider sits behind Streams,
or how their session refresh works, or what their instrument-token space
looks like. The gateway takes care of:

- **Underlying session refresh.** The provider's daily access token is
  refreshed server-side. Your key is long-lived.
- **Identifier translation.** Subscribing with UIR ids is enough; the
  gateway translates them to the provider's token space internally.
- **Stale-tick detection.** Each tick carries a monotonic sequence number.
  Late ticks are dropped before they reach your code.
- **Backpressure.** If your client is slow to consume, the gateway pauses
  the stream and re-emits the most recent state once you're ready.
