# Getting started

This guide gets you from zero to your first live tick in under five minutes.
You will need **one API key** and **a terminal**. That's it.

<Callout type="info" emoji="🛂">
  API keys are issued per customer. If you don't have one yet, request access
  from the [admin panel](/) — it takes about 30 seconds to provision.
</Callout>

## What you'll learn

<Steps>

### Get a key

Sign in, click **Create key**, copy the value. Your key is a long-lived bearer
that you keep on the client. You can rotate it at any time.

### Hit search

```bash
curl "https://marketdata.vedpragya.com/api/search?q=RELIANCE" \
  -H "x-api-key: $STREAMS_KEY"
```

### Subscribe to live ticks

```js
import { io } from 'socket.io-client';
const socket = io('https://marketdata.vedpragya.com', {
  extraHeaders: { 'x-api-key': process.env.STREAMS_KEY },
});
socket.on('market_data', (tick) => console.log(tick));
```

### You're done

You now have search, snapshot prices, and a live feed. Add watchlist CRUD if
you want persistence; the rest is just UI.

</Steps>

## What you don't have to worry about

A handful of details are intentionally hidden from your code:

- **The underlying provider.** Streams sits between your app and our market
  data vendor. We handle authentication, session refresh, and failover.
- **Identifier translation.** Search returns UIR ids. The live feed accepts
  UIR ids. You never need to know the provider's instrument-token space.
- **Stale-tick bookkeeping.** Each tick carries a monotonic sequence
  number; the platform drops late and duplicate ticks for you.
- **Reconnect storms.** Our WebSocket gateway implements exponential
  backoff and message replay for at-most-once delivery of recent ticks.
