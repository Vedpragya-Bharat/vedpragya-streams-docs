# API Reference

This reference documents every API endpoint in the Vedpragya Streams platform.

## Base URLs

| Environment | REST URL                          | WebSocket URL                       |
| ------------- | --------------------------------- | ----------------------------------- |
| Production    | `https://marketdata.vedpragya.com` | `https://marketdata.vedpragya.com`    |
| Staging       | `https://stg.marketdata.vedpragya.com` | `https://stg.marketdata.vedpragya.com` |

All REST endpoints are prefixed with `/api`. WebSocket uses Socket.IO at the root path.

---

## Authentication

Most endpoints require an API key passed via the `x-api-key` header:

```bash
curl -H "x-api-key: vk_live_xxxxxxxx" \
  "https://marketdata.vedpragya.com/api/stock/vayu/ltp"
```

| Header       | Required | Notes                                      |
| ------------ | -------- | ------------------------------------------ |
| `x-api-key` | Yes*     | Required for most endpoints. See table below. |

*Endpoints marked "Public" below do not require authentication.

## Endpoints by Provider

### Vayu Provider (`/api/stock/vayu`)

Vayu is our primary data provider for Indian equities and derivatives.

| Method | Endpoint                       | Auth    | Description                                    |
| -------| ------------------------------ | -------- | ---------------------------------------------- |
| GET    | `/api/stock/vayu/instruments`   | Required | List all instruments (paginated).                |
| GET    | `/api/stock/vayu/instruments/search` | Required | Search instruments by query.                   |
| GET    | `/api/stock/vayu/instruments/stats`   | Required | Get instrument statistics.                  |
| GET    | `/api/stock/vayu/instruments/:token` | Required | Get single instrument details.                |
| POST   | `/api/stock/vayu/ltp`              | Required | Get LTP snapshot for multiple instruments.  |
| POST   | `/api/stock/vayu/validate-instruments` | Required | Validate a list of instrument UIRs.        |
| GET    | `/api/stock/vayu/health`           | Public   | Health check for Vayu provider.              |

#### GET /api/stock/vayu/ltp

Get the latest price for one or more instruments.

```bash
curl -X POST "https://marketdata.vedpragya.com/api/stock/vayu/ltp" \
  -H "x-api-key: vk_live_xxx" \
  -H "Content-Type: application/json" \
  -d '{"instruments": [334562, 26026, 256265]}'
```

**Request:**
```json
{
  "instruments": [334562, 26026, 256265],
  "provider": "vayu"
}
```

| Field         | Type    | Required | Notes                           |
| -------------- | ------- | -------- | ------------------------------ |
| `instruments` | int[]   | Yes      | Array of UIR ids (max 500).    |
| `provider`   | string  | No       | `vayu`, `falcon`, `drift`, `atlas`. |

**Response:**
```json
{
  "success": true,
  "data": {
    "334562": {
      "last_price": 2347.10,
      "ohlc": { "o": 2331.50, "h": 2350.00, "l": 2325.40, "c": 2341.10 },
      "change": 6.00,
      "pchange": 0.26,
      "volume": 12403201,
      "avg_price": 2338.42,
      "ts": "2026-06-06T05:15:00.000Z"
    },
    "26026": {
      "last_price": 23416.55,
      "ohlc": { "o": 23480.10, "h": 23510.40, "l": 23350.00, "c": 23466.65 },
      "change": -50.10,
      "pchange": -0.21,
      "volume": 0,
      "avg_price": null,
      "ts": "2026-06-06T05:15:00.000Z"
    }
  },
  "timestamp": "2026-06-06T05:15:00.000Z"
}
```

| Field        | Type    | Description                             |
| -------------| ------- | --------------------------------------- |
| `last_price` | number  | Most recent trade price.               |
| `ohlc`       | object  | Open, High, Low, Close.                 |
| `change`     | number  | Absolute change from previous close.    |
| `pchange`    | number  | Percentage change.                     |
| `volume`     | integer | Trading volume.                         |
| `avg_price`   | number? | Volume-weighted average price.         |
| `ts`         | string  | Timestamp (ISO-8601).                  |

## Search API

### GET /api/search

Search for instruments by symbol or name. **This endpoint is public (no auth required).**

```bash
curl "https://marketdata.vedpragya.com/api/search?q=RELIANCE&limit=10"
```

**Query Parameters:**

| Parameter | Type    | Required | Notes                              |
| ----------| ------- | -------- | ---------------------------------- |
| `q`       | string   | Yes       | Search query (symbol or name).        |
| `limit`   | int     | No       | Max results (default: 10, max: 50).  |
| `exchange`| string  | No       | Filter by exchange.                |
| `type`    | string  | No       | Filter by instrument type.          |

**Response:**

```json
{
  "success": true,
  "data": [
    {
      "id": 334562,
      "canonicalSymbol": "NSE:RELIANCE",
      "wsSubscribeUirId": 334562,
      "symbol": "RELIANCE",
      "name": "RELIANCE",
      "exchange": "NSE",
      "segment": "NSE",
      "instrumentType": "EQ",
      "assetClass": "equity",
      "isDerivative": false,
      "lotSize": 1,
      "tickSize": 10,
      "priceStatus": "stale",
      "last_price": null,
      "change": null,
      "pchange": null,
      "streamProvider": "falcon",
      "logo_url": "https://..."
    }
  ],
  "timestamp": "2026-06-06T05:15:00.000Z"
}
```

| Field             | Type     | Description                             |
| ------------------| -------- | --------------------------------------- |
| `id`             | integer  | **UIR id** — use for subscriptions.    |
| `canonicalSymbol`| string   | Format: `EXCHANGE:SYMBOL`.              |
| `wsSubscribeUirId` | integer | Same as `id`. Use for WS subscription. |
| `symbol`         | string   | Trading symbol.                         |
| `name`           | string   | Instrument name.                       |
| `exchange`       | string   | Exchange (NSE, BSE, MCX, etc).         |
| `instrumentType`| string   | EQ, IU, FUT, CE, PE, etc.               |
| `assetClass`     | string   | equity, derivative, etc.               |
| `priceStatus`    | string   | live, stale, closed, halted.           |
| `last_price`     | number?  | Last traded price.                    |
| `streamProvider` | string   | Data source: falcon, vayu, drift.      |

### Subscribing to Market Data

```js
// Subscribe to instruments
socket.emit('subscribe', {
  channel: 'market_data',
  instruments: [334562, 26026],     // UIR ids from search
});
```

**Subscribe Event Fields:**

| Field         | Type    | Required | Description                        |
| --------------| --------| -------- | --------------------------------- |
| `channel`    | string  | Yes      | Use `market_data`.               |
| `instruments`| int[]   | Yes      | Array of UIR ids.               |
| `provider`   | string  | No       | `falcon`, `vayu`, `drift`, `atlas`. |

---

### Receiving Ticks

```js
// Market data tick
socket.on('market_data', (tick) => {
  console.log(tick);
});
```

**Tick Payload:**

```json
{
  "uirId": 334562,
  "instrumentToken": 334562,
  "last_price": 2347.10,
  "ohlc": { "o": 2331.50, "h": 2350.00, "l": 2325.40, "c": 2341.10 },
  "change": 6.00,
  "pchange": 0.26,
  "volume": 12403201,
  "avg_price": 2338.42,
  "bid": 2345.50,
  "ask": 2348.70,
  "oi": null,
  "ts": "2026-06-06T05:15:00.000Z"
}
```

| Field          | Type    | Description                          |
| ---------------| ------- | ---------------------------------- |
| `uirId`       | integer | UIR id.                          |
| `instrumentToken` | integer | Same as uirId for compatibility. |
| `last_price`  | number  | Last traded price.                |
| `ohlc`       | object  | OHLC data.                       |
| `change`     | number  | Absolute change.                |
| `pchange`    | number  | Percentage change.               |
| `volume`     | integer | Volume.                         |
| `avg_price`  | number? | VWAP.                          |
| `bid`        | number? | Best bid price.                 |
| `ask`        | number? | Best ask price.                |
| `oi`         | integer?| Open interest (derivatives). |
| `ts`        | string  | Timestamp.                     |

### Subscription Confirmation

```js
socket.on('subscription_confirmed', (ack) => {
  console.log(ack);
});
```

```json
{
  "channel": "market_data",
  "instruments": [334562, 26026],
  "included": [334562],
  "unresolved": [],
  "forbidden": []
}
```

| Field        | Type   | Description                              |
| -------------| -------| -------------------------------------- |
| `channel`   | string| Channel name.                         |
| `instruments` | int[]| Requested UIR ids.                |
| `included`  | int[]| Successfully subscribed UIR ids.       |
| `unresolved`| int[]| UIR ids not found in the system.       |
| `forbidden` | int[]| UIR ids not allowed for this API key.  |

### Error Events

```js

  console.error(err);
});
```

```json
{
  "error": "invalid_instrument",
  "message": "UIR 999999 not found",
  "code": 404
}
```

## Error Codes

| Code | Error                    | Description                                |
| -----| ------------------------| ------------------------------------------ |
| 400  | `invalid_request`        | Malformed request body.                     |
| 401  | `unauthorized`         | Missing or invalid API key.              |
| 403  | `forbidden`            | API key doesn't have permission.           |
| 404  | `not_found`            | Resource not found.                        |
| 429  | `rate_limited`         | Too many requests.                        |
| 500  | `internal_error`        | Server error.                             |

---

## Status Codes

See [priceStatus values](/search/response#when-pricestatus-matters):

- `live` — Active streaming
- `stale` — Has quotes but not streaming
- `closed` — Market closed
- `halted` — Trading paused
- `pre_open` — Pre-market

<Callout type="info" emoji="💡">
  Always check `priceStatus` before displaying prices. Don't show stale data
  as if it were live.
</Callout>
